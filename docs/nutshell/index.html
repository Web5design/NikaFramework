Document describes how quickly install and run sample application.

Install

Go to NikaFramework website and download the latest version of SDK. After extracting, you have to get the following structure:

data - JSON stub for testing purpose

localization - tool to helps localize text found in xhtml files

nkf4 - main SDK folder

nkf4/build - build system

nkf4/impl - components implementation

nkf4/nkf.js - SDK file

.htaccess - need for correct working framework

index.template.xhtml - template of index file, which content will be used to render HTML into.

Move all files to any Web Server, for example Apache. To get working ensure that 

AllowOverride All

added to Apache config file in appropriate place.

Get running first application

Build Application

Build System for NKF written on NodeJS. So you need it to build application. Also there other dependencies. So for installation please read README file in build directory.

After all required software is installed, we can move further. In build directory there is a file called include.list. There are only two lines: 

nkf.js

;impl/app.js

It means, that for build it will include only NKF SDK. impl/app.js is commented, as at the beginning of line we have ; So lets build our application. Go to build folder and execute

node build.js

In out folder should appear two files: merged.css and merged.js. But we shouldn't care about them. In root directory should appear index.xhtmlz file.

index.xhtmlz - generated file by NKF Build System which contains all resources, including JavaScript, XHTML, CSS and any other binary resources such as images, video as base64. For more detail information see Web Resource Bundle

Now hit http://localhost in browser. Yes, nothing will happend. We need to write more code to get it works.

impl/components/widget/Login/logic/Login.js: 

(function() {
  "use strict";

  var ns = "nkf.impl.components.widget";
  var self = $.namespace(ns);

  extendClass(Login, nkf.core.components.component.WidgetAbstract);
  Login.className = "Login";

  // --------------------------------------------------------------------
  // Private methods
  // --------------------------------------------------------------------

  function Login() {
    this.className = Login.className;

    this.getRenderedDOM = function() {
      return domComponent;
    };

    function constructor() {
      init();
    }

    function init() {
      addEventListener();
    }

    function addEventListener() {
      domComponent.find(":submit").click(function(e) {
        e.preventDefault();

        var username = domComponent.find("#login").val().trim();
        var password = domComponent.find("#password").val().trim();

        if (username && password) {
          //Do login, if any credentials were entered
          $.cookie("isLogin", true, {path: "/"});

          $(document).trigger("nkf.core.Controller", {
            type: nkf.def.events.type.make,
            name: "load",
            data: {
              init: true
            }
          });
        } else {
          alert("Please enter login and password!");
        }

      });
    }

    // --------------------------------------------------------------------
    // Private variables
    // --------------------------------------------------------------------

    var _this = this;

    var domComponent = this._getComponent();

    var $ComponentManager = nkf.core.components.ComponentManager.getInstance();

    constructor.call(this);
  }

  $.extend(self, {
    Login: Login
  });

})();

Configure NKF

You can easy change default settings of NKF by redefining them.

Examples:

nkf.conf.defaultNonLoginPage = "AccessDenied";

Add this line in app.js before trigger Controller, rebuild application, logout and refresh browser page. You should find that URL changed to /#path=AccessDenied and JavaScript exception threw, as we don't have that class defined.

You can also change render layout and page components render selector:

nkf.conf.render.layout.selector = "#container";

in that case instead of section tag page component will render into #container selector.

All settings that could be configured you can find at conf.json file.
